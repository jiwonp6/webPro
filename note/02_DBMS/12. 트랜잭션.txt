◎ 트랜잭션
1. 트랜잭션
	: SQL 명령문들을 하나의 논리적인 작업 단위로 처리하는 것,
	 데이터 일관성 유지 + 안정적으로 데이터 복구
		- 명령어집합이 정상적으로 처리 -> 정상 종료
		- 명령어집합 중 하나의 명령어라도 에러 -> 전체 취소
2. 명령어
① COMMIT; 
	: 모든 작업들을 정상적으로 처리하겠다고 확정하는 명령어, 
	  변경된 내용을 모두 영구 저장, 하나의 트랜잭션 과정 종료

② SAVEPOINT;
	: ROLLBACK될 지점 선정
③ ROLLBACK;
	: 현재의 트랜잭션 작게 분할, 작업 취소 이전의 COMMIT지점까지 복구
* COMMIT&ROLLBACK 
	-> 데이터 무결성 보장,
	    영구적 변경 전 데이터의 변경 사항 확인 가능,
	    논리적으로 연관된 작업 그룹화
3. 문법
① COMMIT; 
	: COMMIT; 
② SAVEPOINT;
	: SAVEPOINT 라벨명;
③ ROLLBACK;
	: ROLLBACK [TO 라벨명];
