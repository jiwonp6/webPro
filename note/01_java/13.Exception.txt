◎ 예외처리(Exception)
1. 예외
	: 사전에 문제 발생 시 대응방안 프로그래밍하는 것
	* 에러
	- 문법에러
		: 문법적으로 나타는 에러
	- 실행에러
		- 논리에러
			: 프로그래밍 자체로 에러
		- 시스템에러
			: 프로그래머 의지 상관 없이 나타나는 에러
		- 예외사항
			: 프로그램 정상작동 과정의 에러
2. 예외처리의 필요성
	: 한 부분에서 예외 발생하더라도 프로그램 동작되도록 함
3. try-catch
	* 문법
		: try {
	    		문제가 발생할 수 있는 로직;

		}catch(익셉션타입 익셉션변수) {
			그 익셉션을 처리하는 대처방안;

		} finally {
			익셉션 발생 여부와 상관없이 가장 마지막 실행될 명령문;
		}

4. throws
	: 나를 호출한(실행시키는) 쪽으로 예외 던져버리는 방식
